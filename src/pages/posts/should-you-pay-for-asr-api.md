做語音辨識有兩條路：

1. **自己跑模型**：下載開源模型，在本地或自己的伺服器跑
2. **買 API**：把音頻丟給雲端服務，拿回文字

買 API 看起來很省事，但真的值得嗎？

## 主流的語音辨識 API

### Google Cloud Speech-to-Text

Google 的雲端語音辨識服務。

- **價格**：$0.006/15 秒（標準），$0.009/15 秒（加強版）
- **優點**：辨識品質好，支援 125+ 種語言
- **缺點**：要綁 GCP，計費複雜

換算一下：1 小時音頻大約 $1.44 - $2.16。

### Azure Speech Services

微軟的語音服務。

- **價格**：$1/小時（標準），$1.40/小時（客製化）
- **優點**：跟 Microsoft 365 整合好，企業愛用
- **缺點**：Azure 的介面很難用

### AWS Transcribe

亞馬遜的語音轉文字服務。

- **價格**：$0.024/分鐘 = $1.44/小時
- **優點**：跟 AWS 生態系整合，有批次處理功能
- **缺點**：即時串流要另外加錢

### OpenAI Whisper API

OpenAI 提供的 Whisper 模型 API。

- **價格**：$0.006/分鐘 = $0.36/小時
- **優點**：便宜，品質好（就是 Whisper 模型）
- **缺點**：只有非串流，不支援即時辨識

### Groq

Groq 是一個主打「快」的 AI 推理平台，也有提供 Whisper API。

- **價格**：$0.111/小時（比 OpenAI 貴，但快很多）
- **優點**：超快，延遲極低
- **缺點**：目前還在早期，穩定性未知

### 訊飛

中國的語音辨識龍頭，中文辨識很強。

- **價格**：依方案不同，有免費額度
- **優點**：中文辨識品質頂尖
- **缺點**：資料會傳到中國伺服器，隱私考量

## 價格比較

假設每天處理 1 小時音頻，一個月 30 小時：

| 服務 | 單價/小時 | 月費 |
|------|----------|------|
| OpenAI Whisper | $0.36 | $10.8 |
| AWS Transcribe | $1.44 | $43.2 |
| Google Cloud | $1.44 - $2.16 | $43.2 - $64.8 |
| Azure | $1.00 - $1.40 | $30 - $42 |
| Groq | $0.111 | $3.33 |

Groq 看起來最便宜？但它只有 Whisper，而且是新服務，穩定性要觀察。

## API 的隱藏成本

價格表只是一部分。用 API 還有這些成本：

### 1. 網路延遲

音頻要上傳到雲端，結果要下載回來。

一段 10 秒的音頻（16kHz, 16-bit）大約 320KB。上傳 + 下載 + 處理，最快也要 1-2 秒。

自己跑模型的話，同樣的音頻可能 0.1-0.3 秒就處理完了。

### 2. 網路依賴

沒網路就不能用。

對桌面應用來說，這是硬傷。使用者可能在飛機上、在沒網路的地方。

### 3. 隱私問題

音頻會傳到第三方伺服器。

如果辨識的內容涉及隱私（會議記錄、醫療對話），使用者可能不願意。

### 4. 廠商鎖定

綁定特定雲端平台後，換平台的成本很高。

API 格式不同、計費方式不同、功能不同，遷移很痛苦。

## 什麼時候該用 API？

API 不是不能用，而是要看場景。

**適合用 API 的情況**：

- 快速原型驗證，還不確定產品方向
- 使用量很小，自建不划算
- 需要超多語言支援，自己訓練模型太麻煩
- 團隊沒有 ML 能力，不想碰模型

**不適合用 API 的情況**：

- 產品需要離線使用
- 有隱私要求
- 使用量大，成本敏感
- 需要低延遲（即時串流）
- 想要完全掌控技術棧

## 我的結論：不需要

對我來說，答案很明確：**不需要花錢買 API**。

原因：

### 1. 開源方案夠好了

Sherpa-ONNX 的辨識品質不輸雲端 API，速度還更快（因為沒有網路延遲）。

我花時間優化過的 FunASR（[Hanzi-ASR](https://github.com/Jeffrey0117/Hanzi_ASR)）也達到很好的效果。

### 2. 離線是剛需

我的產品是桌面應用，使用者不一定有網路。而且語音內容可能涉及隱私，不適合上傳到雲端。

### 3. 長期成本更低

自建的話，一次把模型包進應用，之後就是零成本。

用 API 的話，每個月都要付錢，使用量越大越貴。

### 4. 技術掌控

自己跑模型，可以調參數、換模型、做各種客製化。

用 API 就是黑盒子，出問題只能等廠商修。

---

如果你跟我一樣：做離線應用、在意隱私、想要低延遲，那就不需要花錢買 API。

開源方案已經夠好了。
